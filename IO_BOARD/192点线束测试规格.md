# 192 点线束测试规格

## 1. 归档结论

本规格纳入 `ARTERY/IO_BOARD` 后，192 点线束测试机后续统一按 `AT32F455VET7` 平台继续开发、测试和探索。

`ATMEGA8L` 目录中的旧机照片、说明书和反推资料只作为功能参考，不再作为主控方案。便宜版也不再回到 8 位 MCU；直接复用 ARTERY 现有硬件、工程、调试链路和扫描规划。

## 2. 两种产品形态

### 2.1 单片机独立版

由 `AT32F455VET7` 独立完成：

- CD4051 阵列扫描。
- 线序逻辑判断。
- 断线、短路、错位、多点并线判断。
- 本地按键。
- 本地 LED 七段数码管显示。
- 蜂鸣器/指示灯提示。
- 可选串口打印或简单通讯。

适合低成本、离线、快速检测场景。

### 2.2 树莓派 + 单片机版

由树莓派负责上层应用，`AT32F455VET7` 负责底层扫描：

- AT32：高速扫描、ADC/IO 采样、基础电气判断、返回原始数据或结果。
- 树莓派：产品型号档案、CSV/Excel、SQLite/MySQL、界面、云端上传、打印、远程维护。
- 显示由树莓派侧驱动 LCDM/屏幕，AT32 不直接驱动 LCDM。

适合高配版、联网记录、云端追溯和复杂界面场景。

两种产品应尽量共用同一套扫描硬件、电路接口和底层固件模块。

## 3. 现有 ARTERY 工程基础

当前 `IO_BOARD` 已具备以下基础：

- MCU：`AT32F455VET7`，LQFP100。
- 工程：CMake + Ninja，可在 Mac/Ubuntu 现有环境直接使用。
- 调试/烧录：已有 AT-Link、OpenOCD、pyOCD 工作流。
- 现有 mux 规划：
  - `IO_MUX_IN_A`：64 路。
  - `IO_MUX_IN_B`：64 路。
  - `IO_MUX_OUT_A`：64 路。
  - `IO_MUX_OUT_B`：64 路。
- 理论扫描资源：4 组 x 64 = 256 路，覆盖 192 点有余量。
- 现有代码入口：
  - `inc/io_board.h`
  - `src/io_board.c`
  - `src/main.c`

## 4. 192 点映射建议

为兼容现有 ARTERY mux 架构，推荐将 192 点按 64 点分组：

| 点位范围 | 逻辑分组 | 说明 |
|---|---|---|
| OUT 1-64 | `IO_MUX_OUT_A` | 线束 A 端输出组 A |
| OUT 65-128 | `IO_MUX_OUT_B` | 线束 A 端输出组 B，实际 192 点时可只用部分 |
| IN 1-64 | `IO_MUX_IN_A` | 线束 B 端输入组 A |
| IN 65-128 | `IO_MUX_IN_B` | 线束 B 端输入组 B，实际 192 点时可只用部分 |

若按旧机“两块板每块 96 点”的理解，也可以映射为：

| 旧机概念 | ARTERY 映射 |
|---|---|
| 主控板 96 点 | `OUT_A` 64 点 + `OUT_B` 前 32 点 |
| 从板 96 点 | `IN_A` 64 点 + `IN_B` 前 32 点 |

这样合计为 96 对端子，整机外部端子数为 192。

## 5. 测试对象和数据模型

主要面向双端线束：

- A 端端子编号：`OUT`。
- B 端端子编号：`IN`。
- 标准连接关系来自产品档案。

产品档案字段沿用当前规格要求中的 CSV/Excel 思路：

```csv
OUT,STATE1,IN,STATE2,VALUE,REMARK
1,C,1,C,0.01±2%,
2,C,2,C,0.01±2%,
3,C,3,C,0.01±2%,
```

字段含义：

| 字段 | 含义 |
|---|---|
| `OUT` | A 端输出点编号 |
| `STATE1` | A 端状态，`C` 表示参与连接 |
| `IN` | B 端输入点编号 |
| `STATE2` | B 端状态，`C` 表示参与连接 |
| `VALUE` | 电阻/阈值要求，可先保留 |
| `REMARK` | 备注或扩展字段 |

支持一对一、一对多、多点并线、空脚检测。

## 6. 基础扫描流程

单片机独立版和树莓派版底层扫描保持一致：

```text
1. 关闭所有 mux。
2. 选择一个 OUT 点作为测试源。
3. 输出限流测试信号。
4. 依次扫描所有 IN 点。
5. 记录实际导通列表。
6. 与产品档案比较。
7. 输出 PASS/NG 和错误点位。
```

基础错误类型：

| 类型 | 判断 |
|---|---|
| 断线 | 标准目标 IN 点未导通 |
| 错位 | 导通到非标准 IN 点 |
| 短路 | 一个 OUT 点导通到多个非允许 IN 点 |
| 多点并线 | 按档案允许则 PASS，不允许则 NG |
| 空脚异常 | 标准空脚被检测到导通 |

## 7. 显示与上位通讯接口规划

两种产品形态共用一个“显示/上位通讯接口”硬件资源，但装配和功能不同：

| 产品形态 | 接口连接对象 | 显示职责 | AT32 职责 |
|---|---|---|---|
| 单 MCU 旧板/低成本版 | LED 七段数码管显示模块 | LED 七段数码管显示错误点、当前点、状态 | AT32 通过该通讯口驱动显示模块 |
| 树莓派 + 单片机版 | 树莓派通讯口 | LCDM/屏幕由树莓派驱动 | AT32 只做扫描设备，通过该通讯口与树莓派交换命令和结果 |

因此，硬件上建议把这一路定义为 `COMM_DISPLAY_PORT`：

- 单 MCU 旧板版：`COMM_DISPLAY_PORT` 接 LED 七段数码管驱动板/驱动芯片。
- 树莓派版：`COMM_DISPLAY_PORT` 接树莓派 UART/RS485/CAN 等通讯接口。
- 同一块 PCB 可以通过 0R、电阻位、跳帽或 BOM 装配选择连接目标。
- 固件通过 board profile 选择该口是 `DISPLAY_MODE_LED7SEG` 还是
  `COMM_MODE_RPI_HOST`。

### 7.1 单 MCU 旧板 LED 七段数码管显示

低成本独立版保留类似旧机的 LED 七段数码管简单显示：

```text
[错误点十位][错误点个位]    [当前点十位][当前点个位]
```

显示含义：

- 左侧两位：错位/交叉/异常点提示。
- 右侧两位：当前测试点或目标点提示。
- `00 00`：待机或无错误。
- 可配合 `OK/NG LED` 和蜂鸣器增强提示。

由于两位数码管只能直接显示 `00-99`，若产品需要完整显示 `1-192`，有三种处理方式：

- 使用 3 位 + 3 位 LED 七段数码管。
- 两位数码管分页/闪烁显示百位。

LED 七段数码管不建议直接大量占用 MCU GPIO。优先使用串行/总线式驱动：

| 驱动方式 | MCU 占用 | 说明 |
|---|---:|---|
| UART LED 显示模块 | TX/RX 2 线 | 最符合“复用树莓派通讯口”的方向 |
| SPI/移位寄存器驱动 | CLK/DATA/LATCH 3 线 | 成本低，协议简单 |
| I2C/HT16K33 类驱动 | SCL/SDA 2 线 | 模块化方便，但要确认亮度和供货 |
| MAX7219 类驱动 | SPI 类 3 线 | 常见七段管驱动 |

本项目优先按 UART/串行 LED 显示模块规划，使其能与树莓派通讯口共用同一组
MCU 外设资源和连接器位置。

### 7.2 树莓派版 LCDM 显示

树莓派版不要求 AT32 直接驱动 LCDM。LCDM/屏幕由树莓派侧驱动，AT32 只返回：

- 当前扫描状态。
- PASS/NG。
- 错误 OUT/IN 点。
- 原始导通矩阵或压缩结果。
- 设备自检状态。

树莓派收到这些数据后负责界面显示、数据库、打印和云端上传。

## 8. 树莓派通讯模式

树莓派版建议把 AT32 作为扫描设备。通讯内容分两层：

### 8.1 树莓派下发

- 开始测试。
- 产品编号/配方编号。
- 点位范围。
- 阈值参数。
- 调试命令，如单点输出、单点读取、全扫描。

### 8.2 AT32 返回

- 当前扫描状态。
- PASS/NG。
- 错误类型。
- 错误 OUT/IN 点。
- 原始导通矩阵或压缩结果。
- 设备自检状态。

物理层优先级：

1. TTL UART：样机最简单，适合板内连接。
2. RS485：设备线较长时更稳。
3. CAN：多设备或强抗干扰场景。

同一物理位置在单 MCU 旧板版中作为 LED 七段数码管通讯口使用；在树莓派版中作为
树莓派通讯口使用。两者不同时装配，避免 AT32 同一外设同时承担两个角色。

## 9. 电气注意事项

AT32F455 是 3.3V MCU，后续硬件必须统一处理电平：

- MUX 选型固定为 74HC4051，并按 3.3V 供电；AT32 的地址线和使能线可直接驱动
  74HC4051 控制脚，不需要 TTL 电平转换。
- 不建议 74HC4051 用 5V 供电再由 AT32 3.3V GPIO 直接控制，因为 3.3V 不保证满足
  5V HC CMOS 输入高电平裕量。
- 3.3V 供电的 74HC4051 模拟通道信号不能超过其供电轨；若测试激励仍使用 5V，必须在
  进入 74HC4051/AT32 前做限流、分压、钳位或改成独立耐 5V 前端。
- 若模拟测试端使用 5V/1mA，ADC 输入必须分压、限流、钳位，不能把 5V 直接送入 AT32。
- 若扫描逻辑统一 3.3V，测试阈值和抗干扰能力要重新验证。
- 外部测试点必须串保护电阻，建议每点 1k 起步。
- 线束治具接口应预留 ESD/TVS 位置。

## 10. 后续开发方向

优先顺序建议：

1. 在 `IO_BOARD` 中建立 192 点逻辑映射表。
2. 做单点 mux 输出/输入验证。
3. 做 64 点小范围扫描。
4. 扩展到 96 对端子，即外部 192 点。
5. 增加产品档案比较逻辑。
6. 增加旧板 LED 七段数码管显示接口。
7. 增加树莓派通讯协议，并由树莓派驱动 LCDM/屏幕。
8. 增加数据库、云端、打印等高配功能。
