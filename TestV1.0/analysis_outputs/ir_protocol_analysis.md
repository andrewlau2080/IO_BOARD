# 电线测试机-标签打印机红外通讯波形分析
## 解析基准
- Saleae 文件采样率按 16 MHz 换算。
- 包络定义：`MARK` 为红外载波打开，`SPACE` 为红外载波关闭。MCU 发射时按 MARK/SPACE 表驱动 38-42 kHz 载波即可。
- 由于原始 `.logicdata` 为 Saleae 旧版私有格式，本文时序由保存的 Edge Finder 边沿索引还原，包络边界误差约小于 0.12 ms，满足毫秒级发包参考。

## 通道判定
有连接打印机文件中 Edge Finder 数据块顺序为：`打印接收RX`、`打印发射TX`、`测试发射TX`、`测试接收RX`。
无连接打印机文件中未出现 `测试发射TX` 数据块，仅有打印机周期发射/接收侧记录。

## 载波估计
- 打印机发射载波：半周期约 11.938 us，频率约 41.88 kHz。
- 测试机发射载波：半周期约 12.900 us，频率约 38.76 kHz。

## 工作步骤与关键时序
1. 打印机持续周期轮询：周期约 203.147 ms，每帧约 36.693 ms。
2. 测试机未遇到打印机/未满足打印条件时，测试机 TX 不发射；RX 仅等待红外输入。
3. 有连接时，测试机接收到打印机轮询帧；打印机 TX 到测试机 RX 延迟约 0.215 ms。
4. 在触发帧内，测试机约在轮询帧开始后 24.786 ms 开始发射响应。
5. 测试机响应 TX 持续约 2151.005 ms。
6. 打印机接收测试机响应后执行打印，并暂停自身轮询约 2.000 s。
7. 打印机恢复轮询/确认后，测试机应清除本次测试通过锁存，等待下一次测试。

补充判定：普通轮询帧、触发测试机响应的轮询帧、打印后恢复的第一帧三者包络一致，均为下文 `printer_poll_tx` 这组 71 段码。也就是说，目前波形里没有看到独立的不同 ACK 包；测试机可把“打印机在测试机响应后重新出现有效轮询帧”作为清除本次 `print_pending` 的确认条件。

## MCU 实现建议
- 发射侧不要按每个载波边沿查表；用硬件定时器/PWM 产生载波，包络定时器按 CSV 中 MARK/SPACE 开关 PWM。
- 打印机轮询用约 42 kHz 载波；测试机响应用约 39 kHz 载波。若只做接收，包络解码容差建议先设 ±25%。
- 状态机必须保存 `print_pending`。打印机确认/恢复后才清除，避免未打印成功就丢失测试通过状态。
- 多工位场景建议给每台测试机加入防重发窗口：响应完成后等待打印机恢复轮询或超时，再决定清除/重发。

## 文件清单
- `printer_poll_tx_envelope.csv`：打印机发射轮询包完整 71 段，可用于模拟打印机或测试接收逻辑。
- `tester_response_tx_envelope.csv`：测试机响应发射包完整 3321 段，可直接转换为 MCU MARK/SPACE 查表。
- `printer_rx_seen_tester_response_envelope.csv`：打印机接收端看到的测试机响应前段，主要用于核对测试机响应是否能被打印机识别。
